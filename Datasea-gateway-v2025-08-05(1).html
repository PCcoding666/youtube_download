Gateway 调用须知

在使用 Datasea Gateway 之前,请注意以下事项:

  1. 试用申请: 请先向我司申请专属的 user 和 password.
  2. 认证方式: 所有 HTTP(S) 请求必须在请求头中包含 Proxy-Authorization: Basic .此 token 由 'user-region.sessionId:password' 字符串经过 Base64 编码后的结果, region和sessionId由使用者填写, region用于选择线路区域,sessionId用于线路保持和切换,二者均可留空,形如 user.sessionId:password 或者 user-region:password,请注意分割符号分别为'-'和 '.',region取值范围在下面支持区域列表里, sessionId必须是9位正整数.
  3. Gateway 地址: w1.gateway.datasea.network:2002
  4. 线路选择与复用, 通过修改region和sessionId实现:
  • 线路随机切换:每次访问 Gateway,没指定sessionId的情况下(例如 'james:password123456'), 后端都会自动为您切换至一条随机可用线路
  • 线路强制切换: 或者每次指定不一样的9位数sid以强制切换,如果用户对切换线路的要求比较严格,推荐采用这种方式.
  • 指定线路区域: 如需指定线路的区域,请在认证信息中使用 'user-region:password' 的格式,例如:'james-us:password123456',不填写region即为随机区域.
  • 线路复用: 如需复用特定线路,请填写sessionId,例如:'james.987654321:password123456',保持sessionId不变即可复用相同的线路,有效时长10分钟.
  • 同时指定区域和sessionId也是合法的,例如 'james-us.987654321:password123456'.
  • 访问速率控制: 为保证服务稳定性,请将您的访问速率控制在 100 requests/s 以内.

调用示例

命令行调用 (cURL)

curl -x http://w1.gateway.datasea.network:2002 -U <user>:<passwd> https://ipinfo.io/json

请注意: 将 <user> 和 <passwd> 替换为您申请到的实际用户名和密码.
Python 调用 (requests 库)

import requests

# 请替换为您的实际用户名和密码
my_user = "aaaaaa"
my_psw = "bbbbbbbb"

# Gateway 地址
my_gateway0 = "w1.gateway.datasea.network:2002"

# 设置请求头(可选,可根据需要修改)
headers = {
    "User-Agent": "Mozilla/5.0 (Windows NT 10.0; rv:109.0) Gecko/20100101 Firefox/109.0"
}

auth_proxy0 = f"http://{my_user}:{my_psw}@{my_gateway0}"

try:
    resp = requests.get("https://ipinfo.io/json", proxies={"http": auth_proxy0, "https": auth_proxy0}, headers=headers)
    print(f"Successfully connected via {my_gateway0}. Response: {resp.text.strip()}")

except requests.exceptions.RequestException as e:
    print(f"An error occurred: {e}")

服务限制说明

重要提示:

原来申请过的用户名密码均可正常使用

Datasea 不提供中国大陆地区的网络服务,并已对来自中国大陆区域的网络连接进行屏蔽.因此,您需要从中国大陆以外的地区发起网络请求才能成功访问我们的服务

请您自行准备海外服务器或海外客户端,通过这些节点访问Datasea Gateway.
